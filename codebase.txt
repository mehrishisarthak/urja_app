# Urja Codebase Audit

## 1. Project Overview
**Project Name:** Urja
**Purpose:** A sustainable waste management and colony rewards platform that incentivizes residents to track and manage waste effectively. It uses a shared economy model where waste collection is tracked via RFID nodes, and rewards are distributed among colony members.

## 2. Technology Stack
- **Frontend:** Flutter (SDK 3.9.0)
- **State Management:** Riverpod (flutter_riverpod)
- **Navigation:** GoRouter (go_router)
- **Backend:** Firebase (Authentication, Cloud Firestore)
- **IoT Firmware:** C++/Arduino for ESP32
- **UI/UX:** Google Fonts, Lottie Animations, Smooth Page Indicator
- **Hardware:** ESP32, MFRC522 RFID module

## 3. Architectural Design
The project follows a **Feature-First** architecture, separating concerns into distinct business domains:

- **Core Layer (`lib/core/`):** Contains application-wide logic, routing, themes, and shared widgets.
- **Features Layer (`lib/features/`):**
  - `authentication/`: Handles user login, registration (including Google Sign-In), and profile management.
  - `colony_ledger/`: Manages the dashboard, reward calculations, and coin redemption logic.
  - `waste_tracking/`: Manages colony discovery, connection, and real-time tracking.
- **Data Layer:** Uses Firestore for real-time synchronization and atomic operations.
- **State Management:** Utilizes Riverpod Notifiers and Providers to manage business logic and UI state reactively.

## 4. Key Features
1. **Intelligent Onboarding:** A multi-stage Finite State Machine (FSM) that guides users through splash screens, carousels, authentication, and colony setup.
2. **Shared Reward System:** Residents earn 'coins' based on colony-wide activity. A unique 'Base Balance' logic ensures users only earn from activity occurring after they join.
3. **IoT Real-time Tracking:** Hardware nodes (ESP32) scan RFID tags to instantly credit the colony's account in Firestore via atomic increments.
4. **Partner Marketplace:** Integrated system for redeeming coins for vouchers from partners like Amazon, Swiggy, and Paytm.
5. **Waste Schedule:** A dynamic calendar for 'Wet' and 'Dry' waste collection days.

## 5. Security & Data Management
- **Firestore Rules:** Implements strict security rules to ensure only authorized increments are allowed and to protect sensitive colony data.
- **Atomic Operations:** Uses Firestore's `increment` operator for reliable coin counting across multiple hardware nodes and users.

## 7. IoT Integration Details
The hardware component (`esp32/`) is designed to:
1. Connect to Wi-Fi.
2. Initialize the MFRC522 RFID reader.
3. Upon scanning a valid tag (bin or resident ID), perform a server-side atomic increment in the colony's Firestore document.
4. Provide visual/auditory feedback (LED/Buzzer) upon successful credit.
